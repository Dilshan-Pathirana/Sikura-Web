Software Requirements Specification (SRS)

Project: Web-based Operation Video Library
Revision: 1.0
Method: External video links (Google Drive + YouTube), no server video storage

1. Introduction
1.1 Purpose

This document defines the functional, technical, and design requirements for a web application that allows users to browse categorized operation videos and administrators to manage the content.
Videos are not stored on the server. Instead, admins paste video links (Google Drive or YouTube).

1.2 Objectives

Extremely low hosting cost

Fast deployment

Modern UI/UX

Scalable content structure

Secure admin panel

1.3 Scope

The system will allow:

Public users to browse categories and watch videos

Admins to manage categories and operations

Admins to paste Google Drive or YouTube video links

Responsive modern interface

Accessible design

2. System Overview
2.1 Architecture
Next.js full-stack app
MongoDB Atlas database

External video storage (Drive/YouTube)


No Express server required.

4. Video Handling Requirements
4.1 Storage Policy

Videos are NOT stored on server.

Admin pastes URL from:

Google Drive

YouTube

System stores only:

videoUrl

4.2 Google Drive Handling

Admin paste example:

https://drive.google.com/file/d/FILE_ID/view


System converts to:

https://drive.google.com/file/d/FILE_ID/preview


Video rendered via iframe.

Drive video must be:

public

“Anyone with link”

4.3 YouTube Handling

Admin paste:

https://youtube.com/watch?v=ID


Converted to embed:

https://www.youtube.com/embed/ID
